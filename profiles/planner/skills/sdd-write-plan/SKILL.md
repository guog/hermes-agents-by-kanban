# Write PLAN

1. Call `kanban_show()` and verify the merged SPEC commit and project checkout.
2. Read repository rules, architecture and relevant code before deciding. Record only evidence needed for the design.
3. Write `plan.md` plus only necessary research/data-model/contracts/quickstart artifacts.
4. Cover architecture, interfaces, data changes, compatibility/migration, observability, security, testing, rollback and risks. Trace decisions to SPEC requirements.
5. Do not invent requirements or final Task IDs. Block on a material scope decision.
6. Reconcile branch/MR, validate artifacts, and complete with MR/head, paths and verification; never review or merge.
